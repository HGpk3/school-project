# Стек и план

## Почему выбран именно такой стек

Для школьной демонстрации с флешки нужен максимально надёжный вариант:

- `HTML` — разметка интерфейса.
- `CSS` — визуальный стиль, зелёная палитра, адаптивность.
- `JavaScript` — логика игры, очки, раунды, финал, журнал, экспорт протокола.

### Причины выбора

- работает офлайн;
- не требует установки зависимостей;
- запускается обычным двойным кликом по `index.html`;
- легко переносится на флешке;
- меньше риск поломки на чужом компьютере.

## План реализации

1. Подготовить офлайн-структуру проекта.
2. Описать контекст, ограничения и правила использования.
3. Сделать удобный интерфейс для 3 команд.
4. Реализовать игровое табло с вопросами по географии России.
5. Добавить механику специальных вопросов:
   - `Аукцион`;
   - `Кот в мешке`.
6. Реализовать финальный раунд:
   - удаление тем;
   - ставки;
   - фиксация ответов;
   - подсчёт итогов.
7. Добавить журнал событий и экспорт протокола.
8. Проверить сценарии запуска без интернета.

## Архитектурное решение

Приложение сделано как один автономный фронтенд без сборки. Это сознательное решение ради переносимости и надёжности.
