# Своя игра: География России

Офлайн-прототип школьной игры в формате «Своя игра» на тему географии России.  
Проект сделан так, чтобы его можно было запустить с флешки на любом обычном школьном компьютере через браузер.

## 1. Главное о проекте

Что умеет игра:

- 2 игровых раунда с табло вопросов;
- 3 команды;
- ручное изменение названий команд;
- автоматический подсчет очков;
- ручная корректировка очков на случай спорных моментов;
- механики `Обычный вопрос`, `Аукцион`, `Кот в мешке`;
- финальный раунд со ставками;
- журнал событий;
- экспорт текстового протокола;
- автосохранение текущей партии в браузере.

Почему выбран такой формат:

- не нужен интернет;
- не нужны установки и зависимости;
- можно открыть прямо с флешки;
- меньше риск, что что-то не запустится в школе.

## 2. Как запустить

1. Скопируй всю папку проекта на флешку.
2. На нужном компьютере открой папку проекта.
3. Запусти файл `index.html`.
4. Игра откроется в браузере.

Важно:

- копировать нужно именно всю папку, а не только `index.html`;
- интернет не нужен;
- `Node.js`, `npm` и сборка не нужны.

## 3. Как играть

### Перед началом

1. Открой игру.
2. В левой колонке введи названия трёх команд.
3. При необходимости назначь команду, у которой сейчас право выбора вопроса.

### Основной раунд

1. Нажми на стоимость вопроса на табло.
2. Откроется карточка вопроса.
3. Ведущий слушает ответ и отмечает результат кнопками:
   - `Верно`
   - `Неверно`
   - `Никто не ответил`
4. После правильного ответа право выбора переходит к ответившей команде.

### Аукцион

Если открывается `Аукцион`:

1. Ведущий выбирает команду, выигравшую торги.
2. Задает ставку.
3. Отмечает `Ответ верный` или `Ответ неверный`.

### Кот в мешке

Если открывается `Кот в мешке`:

1. Команда, открывшая вопрос, выбирает соперника.
2. Назначает стоимость вопроса.
3. Ведущий отмечает, ответил соперник верно или нет.

### Финал

1. После окончания второго раунда нажми `Перейти к финалу`.
2. Команды по очереди убирают темы.
3. Когда останется одна тема, команды делают ставки.
4. Затем ведущий открывает финальный вопрос и отмечает итог ответа для каждой команды.
5. После этого нажимается `Подвести итоги`.

## 4. Как считается счёт

### Обычный вопрос

- правильный ответ: команда получает стоимость вопроса;
- неправильный ответ: команда теряет стоимость вопроса.

### Аукцион

- правильный ответ: команда получает сумму ставки;
- неправильный ответ: команда теряет сумму ставки.

### Кот в мешке

- выбранная команда отвечает на назначенную стоимость;
- при верном ответе получает эту сумму;
- при ошибке теряет эту сумму.

### Финал

- каждая команда ставит сумму в пределах своего текущего счёта;
- при верном ответе ставка прибавляется;
- при неверном ответе или отсутствии ответа ставка вычитается.

## 5. Кнопки и элементы интерфейса

### Верхняя панель

- `Скачать протокол` — сохраняет текстовый итог игры;
- `Новая игра` — полностью сбрасывает текущую партию.

### Блок команд

- поле с названием команды — можно переименовать;
- `Передать выбор` — назначает команду, которая выбирает следующий вопрос;
- `+ Очки` / `- Очки` — ручная корректировка счёта.

### Блок игры

- показывает текущий раунд;
- показывает, сколько вопросов осталось;
- показывает, у какой команды сейчас ход;
- даёт кнопку перехода к следующему раунду или к финалу.

### Журнал

Показывает последние действия по ходу партии:

- открытие вопроса;
- правильные и неправильные ответы;
- переходы между раундами;
- действия в финале;
- итог победителя.

## 6. Автосохранение

Игра сохраняет состояние в `localStorage` браузера.

Это значит:

- если случайно обновить страницу, партия обычно сохранится;
- если нажать `Новая игра`, старое состояние будет заменено новым;
- на другом компьютере сохранений не будет, если открыть проект впервые.

## 7. Экспорт протокола

Кнопка `Скачать протокол` создает файл `svoya-igra-protokol.txt`.

В него попадают:

- название игры;
- дата экспорта;
- итоговые места команд;
- количество правильных и неправильных ответов;
- журнал действий.

Это удобно, если нужно:

- отчитаться перед учителем;
- сохранить результаты;
- распечатать ход игры.

## 8. Структура проекта

- `index.html` — основная HTML-разметка приложения;
- `styles.css` — оформление, цвета, адаптивность;
- `app.js` — вся игровая логика;
- `docs/STACK_AND_PLAN.md` — объяснение выбранного стека и плана;
- `docs/PROJECT_CONTEXT.md` — проектный контекст и ограничения;
- `WORKLOG.md` — краткий журнал работы.

## 9. Как устроен код

### `index.html`

Содержит каркас страницы:

- верхнюю панель;
- левую боковую колонку;
- игровое табло;
- панель вопроса;
- панель финала.

### `styles.css`

Отвечает за:

- зеленую цветовую тему;
- карточки и кнопки;
- адаптацию под разные размеры экрана;
- визуальные состояния активных элементов.

### `app.js`

Главный файл проекта. В нём находятся:

- база вопросов `GAME_DATA`;
- текущее состояние игры `state`;
- функции отрисовки интерфейса;
- функции обработки ответов;
- подсчет очков;
- логика финала;
- журнал событий;
- экспорт протокола.

## 10. Какие комментарии добавлены в код

В код были добавлены комментарии, чтобы было проще разобраться:

- где хранятся вопросы и темы;
- где хранится состояние партии;
- какие функции отвечают за интерфейс;
- какие функции отвечают за логику вопросов;
- как считается счёт;
- как работает финал;
- как устроено сохранение и экспорт.

Комментарии сделаны не на каждую строку, а по смысловым блокам, чтобы код оставался читаемым.

## 11. Как редактировать проект

Если захочется поменять содержимое:

### Заменить вопросы

Открой `app.js` и найди объект `GAME_DATA`.

Там можно изменить:

- названия категорий;
- тексты вопросов;
- ответы;
- стоимости вопросов;
- финальные темы.

### Изменить цвета

Открой `styles.css` и посмотри блок `:root` в самом начале файла.

Там находятся основные цветовые переменные.

### Изменить тексты интерфейса

Основные подписи находятся в `app.js` и частично в `index.html`.

## 12. Ограничения проекта

Это не телевизионная система с кнопками-сигналами и таймерами, а удобный школьный прототип с ведущим.

Поэтому здесь сознательно сделан упор на:

- понятность;
- надежность;
- ручное управление спорными моментами;
- быстрый запуск без подготовки.

## 13. Если что-то пошло не так

### Игра не открывается

- проверь, что рядом с `index.html` лежат `styles.css` и `app.js`;
- попробуй открыть файл в другом браузере.

### Сбился счет

- используй ручную корректировку очков у нужной команды.

### Случайно закрыли страницу

- открой `index.html` снова: игра должна восстановиться из сохранения браузера.

### Нужно начать заново

- нажми `Новая игра`.

## 14. Итог

Проект предназначен для реального школьного показа:

- запускается просто;
- выглядит аккуратно;
- поддерживает 3 команды;
- считает очки;
- подводит итоги;
