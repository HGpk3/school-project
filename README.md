# Своя игра: География России

Офлайн-прототип школьной игры в формате «Своя игра» на тему географии России.

Автор: **Немых Никита Денисович**

Проект запускается прямо с флешки через браузер и не требует интернета, сборки или установки дополнительных программ.

## 1. Что есть в проекте

- стартовое меню с кнопками `Играть`, `Продолжить` и `Настройки`;
- экран подготовки перед началом партии;
- ввод названий трёх команд до старта игры;
- краткая инструкция по правилам перед запуском;
- 2 игровых раунда;
- спецмеханики `Аукцион` и `Кот в мешке`;
- финальный раунд со ставками;
- журнал событий;
- экспорт текстового протокола;
- автосохранение прогресса в браузере;
- встроенные офлайн-звуки интерфейса;
- настройки звука.

## 2. Как запустить

1. Скопируй всю папку проекта на флешку.
2. На школьном компьютере открой папку.
3. Запусти файл `index.html`.

Важно:

- копировать нужно всю папку, а не только один файл;
- интернет не нужен;
- `Node.js`, `npm` и другие инструменты не нужны.

## 3. Сценарий работы

Теперь проект устроен по шагам:

### Шаг 1. Меню

После открытия появляется стартовый экран.

На нём есть:

- `Играть` — перейти к подготовке новой игры;
- `Продолжить` — вернуться к сохранённой партии, если она уже была;
- `Настройки` — открыть управление звуками.

### Шаг 2. Подготовка

Перед началом партии открывается отдельный экран.

На нём:

- вводятся названия команд;
- показываются краткие правила игры;
- после этого нажимается `Начать игру`.

### Шаг 3. Сама игра

После подготовки открывается игровое поле:

- сверху идут карточки команд;
- в центре располагается табло;
- сбоку находятся журнал и сводка по партии;
- вопросы и финал открываются во всплывающих окнах, чтобы табло не сжималось.

## 4. Как играть

### Основной раунд

1. Команда с правом хода выбирает вопрос.
2. Ведущий открывает клетку на табло.
3. После ответа отмечается результат:
   - `Верно`
   - `Неверно`
   - `Никто не ответил`

Если ответ правильный, команда получает очки и право выбора следующего вопроса.

### Аукцион

Если открывается аукцион:

1. Ведущий выбирает команду, выигравшую торги.
2. Устанавливает ставку.
3. Отмечает результат ответа.

### Кот в мешке

Если открывается кот в мешке:

1. Команда, открывшая вопрос, выбирает соперника.
2. Задает стоимость вопроса.
3. Ведущий отмечает, взял ли соперник вопрос.

### Финал

1. После второго раунда нажимается переход к финалу.
2. Команды по очереди убирают темы.
3. Когда остается одна тема, команды делают ставки.
4. Затем открывается финальный вопрос.
5. После отметки ответов подводятся итоги.

## 5. Как считается счёт

### Обычный вопрос

- верный ответ: `+ стоимость вопроса`
- неверный ответ: `- стоимость вопроса`

### Аукцион

- верный ответ: `+ ставка`
- неверный ответ: `- ставка`

### Кот в мешке

- отвечает только выбранная команда;
- при верном ответе получает назначенную сумму;
- при ошибке теряет назначенную сумму.

### Финал

- ставка не может превышать текущий счёт команды;
- верный ответ прибавляет ставку;
- неверный ответ или отсутствие ответа вычитает ставку.

## 6. Звуки и настройки

В проект добавлены встроенные звуки интерфейса.

Они работают без аудиофайлов, потому что создаются прямо в браузере через Web Audio API.

В настройках можно управлять:

- общим включением звука;
- звуками кликов по кнопкам;
- звуками набора текста;
- звуками результата;
- общей громкостью.

Какие сигналы используются:

- короткий звук при нажатии кнопок;
- лёгкий звук при вводе текста;
- отдельный сигнал для успешных действий;
- отдельный сигнал для ошибок;
- мягкий переходный звук для смены этапов.

## 7. Как устроен интерфейс

### Меню

Содержит:

- название игры;
- информацию об авторе;
- быстрый вход в игру;
- доступ к настройкам.

### Экран подготовки

Содержит:

- поля ввода названий команд;
- краткие правила;
- кнопку запуска новой партии.

### Игровой экран

Содержит:

- верхнюю панель с управлением;
- блок команд;
- центральное табло;
- правую панель со сводкой и журналом;
- модальные окна для вопросов и финала.

## 8. Почему интерфейс теперь лучше помещается

Раньше игровая часть была тесной из-за боковой структуры.

Теперь сделано так:

- карточки команд вынесены в отдельную горизонтальную полосу;
- табло занимает основную ширину экрана;
- вопрос и финал открываются во всплывающих окнах;
- боковая часть стала уже и компактнее;
- на меньших экранах блоки автоматически перестраиваются ниже.

Это делает интерфейс удобнее для школьного монитора, ноутбука и показа через проектор.

## 9. Автосохранение

Состояние партии сохраняется в `localStorage`.

Это значит:

- если случайно обновить страницу, прогресс обычно останется;
- кнопка `Продолжить` в меню возвращает к сохраненной партии;
- кнопка `Новая игра` переводит к новому запуску через экран подготовки.

## 10. Экспорт протокола

Кнопка `Протокол` или `Скачать протокол` создает файл:

`svoya-igra-protokol.txt`

В него попадают:

- название игры;
- имя автора;
- дата выгрузки;
- итоговые места команд;
- статистика ответов;
- журнал всей партии.

## 11. Структура файлов

- `index.html` — разметка экранов, меню, модалок и основного интерфейса;
- `styles.css` — оформление, адаптивность, цвета и компоновка;
- `app.js` — логика игры, меню, настройки, звук, финал и экспорт;
- `docs/STACK_AND_PLAN.md` — стек и общий план проекта;
- `docs/PROJECT_CONTEXT.md` — контекст и ограничения;
- `WORKLOG.md` — журнал работы.

## 14. Как редактировать проект

### Поменять вопросы

Открой `app.js` и найди объект `GAME_DATA`.

Там можно изменить:

- категории;
- тексты вопросов;
- ответы;
- типы вопросов;
- финальные темы.

### Поменять автора

Имя автора указано:

- в `index.html` на стартовом экране;
- в `app.js` внутри экспорта протокола.

### Поменять цвета

Открой `styles.css` и измени переменные в блоке `:root`.

### Поменять правила на экране подготовки

Открой `index.html` и найди блок `rules-card`.

## 15. Ограничения

Это не телевизионная система с таймерами и кнопками-сигналами, а удобный школьный прототип с ведущим.

Поэтому акцент сделан на:

- понятности;
- приятном интерфейсе;
- управляемости;
- офлайн-запуске;
- надёжности на школьном компьютере.

## 16. Если что-то пошло не так

### Проект не открылся

- проверь, что рядом с `index.html` лежат `styles.css` и `app.js`;
- попробуй открыть в другом браузере.

### Нет звука

- открой настройки;
- проверь, что включён общий звук;
- увеличь громкость в настройках;
- нажми любую кнопку ещё раз, чтобы браузер разрешил воспроизведение звука после взаимодействия.

### Случайно сбросили этап

- если партия уже шла, можно вернуться через кнопку `Продолжить` в меню;
- если была начата новая игра, старая партия уже заменена новым состоянием.

### Сбился счёт

- используй ручную корректировку `+` и `-` у нужной команды.
